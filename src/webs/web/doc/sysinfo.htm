General Information

Page shows general information about device, such as:

  • Model (including hardware release)
  • Software Version
  • Serial Number
  • Unique Unit Identifier
  • Number of VoIP Ports available
  • Number of Licensed VoIP Ports
  • Modeof Operation (Switch Side (GW) Unit or Extension Side (EX) Unit)

CPU Information

This page shows information about processes, jobs and programs which are currently running on the server.

  • UID - UID column shows who started and owns the process.
  • PID - PID means process identification number.
  • PPID - The PPID (Parent Process ID) column shows which process started that particular process.
  • C - CPU ID
  • STIME - system time
  • TTY - The TTY column shows which terminal the process was started from.
  • TIME - The TIME column shows the total amount of system (CPU) time used by the process so far.
  • CMD - Command which run the process.

Memory

This page shows the total amount of memory, free memory and used (both physical and swapped) memory in the system, as well as the buffers used by the kernel.

  • Total - Amount of total swapped space
  • Used - Amount of swapped space used
  • Free - Amount of swapped space free
  • Shared - Shared memory. A portion of memory capable of being accessed by more than one application at a time.
  • Buffers - Buffers used by the kernel. Temporary storage in memory used to temporarily store information while other information is being processed.
  • Cached - Memory cache is a portion on memory of high-speed static RAM. It is effective because most programs access the same data or instructions over and over.

Disk

This page shows the amount of disk space used by the specified files, and by each directory in the hierarchies rooted at the specified files.

  • Filesystem - Refers to the parts of the entire hierarchy of directories.
  • 1K-blocks - Number of 1K-blocks.
  • Used - Blocks in use.
  • Available - Shows free space on root partition.
  • Use% - Using total disk capacity percent (%).
  • Mounted on - The column of the name of the mount.

Users

Page shows information about the users currently logged on the machine, and their proccesses. The red header shows, in this order, the current time, how long the system has been running, how many users currently logged on, and the system load averages for the past 1, 5, and 15 minutes.

  • User - Shows users which are logged on.
  • TTY - The TTY column shows from which terminal the process was started from.
  • FROM - Shows address form user has logged on.
  • LOGIN@ - Time of initial login.
  • IDLE - The IDLE column shows the time that the computer or device has been powered on but has not been used.
  • JCPU - The JCPU shoes the time used by all processes attached to the respective TTY.
  • PCPU - The PCPU shows the time used by the current procsses, named in the "WHAT" field.
  • WHAT - The WHAT column shows names of current processes.

Network Information

Page shows information about Network Status.

  • Kernel Interface table - Shows statistics for the network interfaces currently configured.
    • Iface - Network interface.
    • MTU - Shows the current MTU.
    • Met - Shows numeric value of MTU interface.
    • RX-OK - Shows how many packets have been received error free.
    • RX-ERR - Shows how many received packets have been damaged.
    • RX-DRP - Shows how many received packets have been dropped.
    • RX-OVR - Shows how many received packets were lost because of overrun.
    • RX-OK - Shows how many packets have been transmitted error free.
    • RX-ERR - Shows how many transmitted packets have been damaged.
    • RX-DRP - Shows how many transmitted packets have been dropped.
    • RX-OVR - Shows how many transmitted packets were lost because of overrun.
    • Flg - Shows the flags that have been set for this interface.
      • B - A broadcast address has been set.
      • L - This interface is a loopback device.
      • M - All packets are received (promiscuous mode).
      • N - Trailers are avoided.
      • O - ARP is turned off for this interface.
      • P - This is a point-to-point connection.
      • R - Interface is running.
      • U - Interface is up.
  • Displays the status of the currently active interfaces.
  • Kernel IP routing table - Displays the kernel routing table and shows numerical addresses.
    • Destination - The destination network or destination host.
    • Gateway - Shows the gateway the routing entry points to. If no gateway is used, an asterisk is printed instead.
    • Genmask - Shows the "generality" of the route.
    • Flags - Shows various flags that describe the route.
      • G - The route uses a gateway.
      • U - The interface to be used is up.
      • H - Only a single host can be reached through the route.
      • D - This is set if the table entry has been generated y an ICMP redirect message.
      • M - This is set if the table entry was modified by an ICMP redirect message.
    • MSS - Default maximum segement size for TCP connections over this route.
    • Window - Default window size for TCP connections over this route.
    • irtt - Initial RTT (Round Trip Time). The kernel uses this to guess about the best TCP protocol parameters without waiting on (possibly slow) answers.
    • Iface - Network interface.
  • Active Internet connections (w/o servers) - Shows active TCP and UDP, and numerical addresses.
    • Proto - Using protocol.
    • Recv-Q - The count of bytes not copied by the user program connected to this socket.
    • Send-Q - The count of bytes not acknowledged by the remote host.
    • Local Address - Address and port number of the local end of the socket.
    • Foregin Address - Address and port number of remote end of the socket.
    • State - The state of the socket.
      • ESTABLISHED - The socket has an established connection.
      • SYN_SENT - The socket is actively attempting to establish a connection.
      • SYN_RECV - A connection request has been received from the network.
      • FIN_WAIT1 - The socket is closed, and the connection is shutting down.
      • FIN_WAIT2 - Connection is closed, and the socket is waiting for a shutdown from the remote end.
      • TIME_WAIT - The socket is waiting after close to handle packets still in the network.
      • CLOSED - The socket is not being used.
      • CLOSE_WAIT - The remote end has shut down, waiting for the socket to close.
      • LAST_ACK - The remote end has shut down, and the socket is closed. Waiting for acknowledgement.
      • LISTEN - The socket is listening for incoming connections.
      • CLOSING - Both sockets are shut down but we still don't have all our data sent.
      • UNKNOWN - The state of the socket is unknown.
  • Shows an ARP table.
    • Address - The IP address to be assigned to the interface.
    • HWtype - Hardware type.
    • HWaddress - Hardwaer address.
    • Flags - C or CMP is possible.
    • Mask - If it's set refers to the subnet mask.
    • Iface - Network interface.
